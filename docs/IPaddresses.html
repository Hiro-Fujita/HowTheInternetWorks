IP Addresses, Packets, and Routing

All the different devices in the world have a unique address made up of numbers; sort of like a phone number or street address. This is called the internet protocol or IP -- the address is then called the IP Address. Computers communicate, for example, by one computer sending a message via IP Address to the receiving computer and requests information. (Accessing a website) Traditional IP Addresses are 32 bit but due to the sheer amount of users (over 4 billion), we are currently in the middle of a multi-transition to a longer IP address format called the IPv6: 128 bits.

A packet is a small segment of a larger message. Data sent over computer networks, such as the Internet, is divided into packets. These packets are then recombined by the computer or device that receives them. The data travels on the internet in a much less direct fashion. Depending on the traffic congestion, road conditions, and other inconveniences, it automatically chooses the best path for the packets to successfully reach the requester.

If a photo you want to send is too big, the computer can break down the picture to different pieces (packets). It is then sent through various routers which chooses the cheapest available path for each piece of data. Another example is Spotify. The transferring of music is done the same way with the use of packets. To be sure you receive all packets, TCP or transmission control protocol manages the sending and receiving of all your data as packets. If all packets of a song are not sent, TCP will not approve of the packets and an error will occur. Spotify will then resend the missing packets to ensure it is all delivered.
